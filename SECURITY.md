# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in shadcn-vue-echarts, please email [security@example.com](mailto:security@example.com) instead of using the issue tracker. Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

We will acknowledge receipt within 48 hours and aim to provide a fix within one week. Please do not disclose the vulnerability publicly until we have issued a fix.

## Supported Versions

We release security updates for:

- The latest major version
- The previous major version (for 6 months after release)

Example:
- v2.x (current) - receives all updates
- v1.x - receives critical security fixes for 6 months after v2.0 release

## Security Best Practices

When using shadcn-vue-echarts:

1. Keep dependencies up to date
2. Monitor security advisories
3. Report suspicious activity immediately
4. Review ECharts security notices (as a peer dependency)

Thank you for helping keep shadcn-vue-echarts secure!
